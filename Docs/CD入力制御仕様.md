# 商品コード(CD)入力制御コンポーネント 仕様書

## 概要

商品コードを入力・検索し、対応する商品名を表示する再利用可能なコンポーネント。
数値入力による直接指定と、モーダルダイアログによる商品名検索の2つの入力手段を提供する。

MUI版・Ant Design版の両方を実装する。

---

## コンポーネント構成

```
┌─────────────────────────────────────────────────────┐
│  商品CD入力                                          │
│  ┌──────────────┐ ┌────┐ ┌──────────────────────┐   │
│  │ 商品コード    │ │検索│ │ 商品名（読み取り専用）│   │
│  │ (数値入力)    │ │ 🔍 │ │                      │   │
│  └──────────────┘ └────┘ └──────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

| 要素 | 説明 |
|---|---|
| 商品コード入力 | 数値入力フィールド。商品コード(number)を直接入力する |
| 検索ボタン | クリックでモーダルダイアログを開く |
| 商品名表示 | 読み取り専用。ヒットした商品名を表示する |

---

## 操作フロー

### フロー1: 商品コード直接入力

```
[商品コード入力] → Enter or フォーカスアウト
    ├── 一致する商品あり → 商品名を表示
    └── 一致する商品なし → エラー状態（赤枠）、商品名はクリア
```

1. ユーザーが商品コード入力フィールドに数値を入力する
2. **Enter押下** または **フォーカスアウト** をトリガーに検索を実行する
3. 商品データ(JSON)から一致するコードを検索する
4. **ヒットした場合**: 商品名表示フィールドに商品名を表示する
5. **ヒットしなかった場合**: 入力フィールドをエラー状態（赤枠）にし、商品名をクリアする

### フロー2: モーダルダイアログによる検索

```
[検索ボタン] → モーダルオープン（全件表示）
    → 商品名を入力するとリアルタイムにテーブルが絞り込まれる
    → 行クリックで確定
    → モーダルクローズ
    → 商品コード入力に値セット、商品名を表示
```

1. ユーザーが検索ボタンをクリックする
2. モーダルダイアログが開く
3. モーダル内の検索フィールドに商品名を入力する
4. 入力に応じてリアルタイムにテーブルが絞り込まれる（インクリメンタルサーチ）
5. ヒットした商品をテーブル形式で一覧表示する（該当なしの場合は「該当する商品がありません」を表示）
6. テーブルの行をクリックで確定する
7. モーダルが閉じ、商品コード入力フィールドに選択した商品コード、商品名表示に商品名がセットされる

## 各要素の詳細仕様

### 商品コード入力フィールド

| 項目 | 仕様 |
|---|---|
| 入力形式 | 数値のみ |
| 確定トリガー | Enter押下 / フォーカスアウト |
| エラー状態 | 存在しないコード入力時に赤枠表示 |
| エラー解除 | 値を変更し始めた時点でエラーを解除する |
| 空入力時 | エラーにしない（商品名もクリア） |

### 検索ボタン

| 項目 | 仕様 |
|---|---|
| 表示 | アイコンボタン（検索アイコン） |
| 動作 | クリックでモーダルダイアログを開く |

### 商品名表示フィールド

| 項目 | 仕様 |
|---|---|
| 状態 | 読み取り専用 (readonly) |
| 表示内容 | 確定した商品の商品名 |
| 未選択時 | 空欄 |

## モーダルダイアログ仕様

### レイアウト

```
┌──────────────────────────────────────┐
│  商品検索                       [×]  │
├──────────────────────────────────────┤
│                                      │
│  ┌────────────────────────────┐     │

│  └────────────────────────────┘     │
│                                      │
│  ┌──────────────────────────────┐   │
│  │ 商品コード │ 商品名          │   │
│  ├────────────┼─────────────────┤   │
│  │     1      │ ボールペン（黒）│   │ ← 行クリックで確定
│  │     2      │ ボールペン（赤）│   │
│  │     3      │ シャープペンシル│   │
│  │   ...      │ ...             │   │
│  └──────────────────────────────┘   │
│                                      │
└──────────────────────────────────────┘
```

### モーダル内動作

| 項目 | 仕様 |
|---|---|
| 検索方式 | 商品名の部分一致・インクリメンタルサーチ（入力に応じてリアルタイム絞り込み） |
| 結果表示 | テーブル形式（商品コード列・商品名列） |
| 該当なし | 「該当する商品がありません」メッセージ表示 |
| 確定方法 | テーブルの行をクリック |
| 初期状態 | 検索フィールド空欄、テーブルは全件表示 |
| 閉じる | ×ボタン / モーダル外クリック / Escキー |

## 商品データ

### データ形式

```ts
interface Product {
  code: number
  name: string
}
```

### データソース

JSONファイル (`src/data/products.json`) に100件程度のデモデータを格納する。

```json
[
  { "code": 1, "name": "ボールペン（黒）" },
  { "code": 2, "name": "ボールペン（赤）" },
  { "code": 3, "name": "シャープペンシル" },
  { "code": 4, "name": "消しゴム" },
  { "code": 5, "name": "修正テープ" },
  ...
]
```

## コンポーネントインターフェース (Props)

```ts
interface ProductCodeInputProps {
  /** 現在の商品コード値 */
  value: number | null
  /** 値変更時のコールバック（商品コードと商品名を返す） */
  onChange: (code: number | null, name: string) => void
}
```

### 使用例

```tsx
const [productCode, setProductCode] = useState<number | null>(null)
const [productName, setProductName] = useState('')

<MuiProductCodeInput
  value={productCode}
  onChange={(code, name) => {
    setProductCode(code)
    setProductName(name)
  }}
/>
```

## MUI版・Ant Design版の実装方針

### MUI版

| 要素 | 使用コンポーネント |
|---|---|
| 商品コード入力 | `TextField` (type="number") |
| 検索ボタン | `IconButton` + `SearchIcon` |
| 商品名表示 | `TextField` (InputProps.readOnly) |
| モーダル | `Dialog` |
| モーダル内テーブル | `Table` / `TableBody` / `TableRow` |
| エラー状態 | `TextField` の `error` prop |

### Ant Design版

| 要素 | 使用コンポーネント |
|---|---|
| 商品コード入力 | `InputNumber` |
| 検索ボタン | `Button` (icon={<SearchOutlined />}) |
| 商品名表示 | `Input` (readOnly) |
| モーダル | `Modal` |
| モーダル内テーブル | `Table` |
| エラー状態 | `InputNumber` の `status="error"` |

## メイン画面への組み込み

既存の `App.tsx` のタブ一覧に「CD入力」タブを追加し、MUI/Antd比較形式で表示する。

### タブ定義への追加

```ts
const tabs = [
  // ...既存のタブ
  { key: 'code', label: 'CD入力', description: '商品コード検索、モーダル検索、エラー制御' },
]
```

### タブコンテンツ

```ts
const tabContent = {
  // ...既存のコンテンツ
  code: { mui: <MuiProductCodeInput ... />, antd: <AntdProductCodeInput ... /> },
}
```

既存の `ComparisonRow` を使い、左にMUI版・右にAnt Design版を横並びで配置する。
各コンポーネントの下に、現在の選択状態（商品コード・商品名）を表示する。

---

## 備考

- コンポーネントは複数画面での再利用を前提とする
- 商品データは静的JSONから読み込み、将来的にAPI化する場合はデータ取得層のみ差し替える想定
- エラー状態は値を編集し始めた時点で解除される
- モーダルを開いた初期状態では全件がテーブルに表示される
