# UI ライブラリ入力コンポーネント比較アプリ 仕様書

## 概要

MUI (Material UI) と Ant Design の入力系コンポーネントを、同一ページ上で横並びに比較できる Web アプリケーション。
各コンポーネントの見た目・バリデーション・カスタマイズ性の違いを実際に操作して確認できる。

---

## 技術スタック

| 項目 | 技術 |
|---|---|
| フレームワーク | React 19 |
| ビルドツール | Vite |
| 言語 | TypeScript |
| UIライブラリ | MUI (`@mui/material`, `@mui/x-date-pickers`) / Ant Design (`antd`) |
| 数値フォーマット | react-number-format（MUI側で使用） |
| 日付ライブラリ | dayjs（MUI・Ant Design共通） |

## コンポーネント別仕様

### 1. 日付入力 (DatePicker)

| 項目 | MUI | Ant Design |
|---|---|---|
| コンポーネント | `@mui/x-date-pickers/DatePicker` | `antd/DatePicker` |
| 日付ライブラリ | dayjs (AdapterDayjs) | dayjs (標準) |

**バリエーション（各ライブラリで以下パターンを並べる）:**

| パターン | 説明 |
|---|---|
| カレンダーあり（デフォルト） | アイコンクリックでカレンダーポップアップ |
| カレンダーなし（手入力のみ） | テキスト入力のみで日付を入力 |
| フォーマット `YYYY/MM/DD` | スラッシュ区切り |
| フォーマット `YYYY-MM-DD` | ハイフン区切り |
| フォーマット `YYYY年MM月DD日` | 日本語フォーマット |
| 範囲制限付き | min/max で入力可能範囲を制限 |
| 初期値あり / なし | 初期値の有無による表示の違い |

**試す内容:**
- 手入力での日付入力と自動フォーマット
- 無効な日付（2月30日など）の入力時の挙動
- カレンダーUIのロケール（日本語）対応
- キーボードだけで入力完了できるか

### 2. 日時入力 (DateTimePicker)

| 項目 | MUI | Ant Design |
|---|---|---|
| コンポーネント | `@mui/x-date-pickers/DateTimePicker` | `antd/DatePicker` (showTime) |

**バリエーション:**

| パターン | 説明 |
|---|---|
| カレンダー+時刻選択あり | デフォルトのフルUI |
| 手入力のみ | カレンダー/時刻ピッカーなし |
| フォーマット `YYYY/MM/DD HH:mm` | 分まで |
| フォーマット `YYYY/MM/DD HH:mm:ss` | 秒まで |
| 時刻のステップ指定 | 15分刻み、30分刻みなど |

**試す内容:**
- 日付部分と時刻部分の入力切替のUX
- 時刻選択UI（MUIのクロック vs Antdのドラム）
- 手入力時の時刻フォーマットの自動補完

### 3. 時刻入力 (TimePicker)

| 項目 | MUI | Ant Design |
|---|---|---|
| コンポーネント | `@mui/x-date-pickers/TimePicker` | `antd/TimePicker` |

**バリエーション:**

| パターン | 説明 |
|---|---|
| 時刻ピッカーあり | デフォルトUI |
| 手入力のみ | ピッカーなし |
| `HH:mm` フォーマット | 24時間制・分まで |
| `HH:mm:ss` フォーマット | 24時間制・秒まで |
| `hh:mm A` フォーマット | 12時間制（AM/PM） |
| ステップ指定 | 15分刻み、30分刻み |

**試す内容:**
- MUIのクロックUI vs Antdのドラムスクロール
- 手入力時の挙動の違い
- キーボード操作のしやすさ

### 4. 数値入力

| 項目 | MUI | Ant Design |
|---|---|---|
| コンポーネント | `TextField` + `react-number-format` | `InputNumber` |
| カンマ区切り | react-number-format で実現 | `formatter`/`parser` で実現 |
| 小数点 | react-number-format の設定 | `precision` プロパティ |

**バリエーション:**

| パターン | 説明 |
|---|---|
| カンマ区切り整数 | `1,234,567` |
| カンマ区切り+小数点 | `1,234.56` |
| 小数点のみ（カンマなし） | `1234.56` |
| パーセント入力 | `85.5%` |
| 通貨入力 | `¥1,234,567` |
| 最小値・最大値制限 | 0〜100 などの範囲制限 |
| マイナス値許可 / 不許可 | 負数の入力可否 |
| ステッパー付き | +/- ボタンで増減 |

**試す内容:**
- カンマ区切り表示と実際の送信値（数値）の違い
- コピー&ペースト時の挙動（カンマ付きテキストを貼り付け）
- 入力中と確定後（onBlur）の表示の違い
- ステッパーの増減幅の設定

### 5. 文字列入力

| 項目 | MUI | Ant Design |
|---|---|---|
| コンポーネント | `TextField` | `Input` |
| バリデーション | 自前 or ライブラリ | `rules` + `Form` |

**バリエーション:**

| パターン | 説明 |
|---|---|
| 基本テキスト | プレーンなテキスト入力 |
| 文字数制限付き | maxLength + 残り文字数表示 |
| パスワード入力 | マスク表示+表示切替 |
| 複数行（TextArea） | テキストエリア |
| prefix/suffix付き | 前後にアイコンやテキスト |
| パターンマスク | 電話番号 `000-0000-0000` 等 |
| disabled / readonly | 非活性・読み取り専用 |
| エラー表示 | バリデーションエラー状態 |

**試す内容:**
- placeholder / label の表示方法の違い（floating label等）
- エラー表示のデザインとタイミング（onChange / onBlur）
- IME入力（日本語入力）時の挙動

### 6. Autocomplete (値マッピング付き)

| 項目 | MUI | Ant Design |
|---|---|---|
| コンポーネント | `Autocomplete` | `AutoComplete` / `Select` (showSearch) |
| 特徴 | フィルタリング・グルーピング等が充実 | シンプルな候補表示 |

**重要: 表示値と送信値のマッピング**

ユーザーが「はい」と表示されている選択肢を選んだ場合、内部値は `2` がサーバーに送信される。

```ts
// 選択肢の定義
const options = [
  { label: 'いいえ', value: 1 },
  { label: 'はい',   value: 2 },
];
// 表示: 「はい」 → 送信値: 2
```

**バリエーション:**

| パターン | 説明 |
|---|---|
| 基本（label/value マッピング） | 表示は `label`、送信は `value` |
| 自由入力許可 (freeSolo) | 候補にない値も入力可能 |
| 自由入力不許可 | 候補からのみ選択 |
| 複数選択 | 複数項目を選択可能 |
| グループ化 | カテゴリ別にグルーピング表示 |
| 非同期データ読み込み | API呼び出し風のローディング表示 |
| カスタム表示 | 選択肢にアイコンや補足情報を表示 |

**試す内容:**
- 入力テキストによる候補絞り込みの挙動
- label/value マッピングが正しく動作するか
- 選択後の表示値と内部値（送信値）の確認
- 日本語入力（IME）での絞り込み

### 7. Select (値マッピング付き)

| 項目 | MUI | Ant Design |
|---|---|---|
| コンポーネント | `Select` | `Select` |
| 特徴 | TextField統合型 | 単体で高機能 |

**重要: 表示値と送信値のマッピング**

Autocompleteと同様、表示ラベルと送信値を分離する。

```ts
const options = [
  { label: '未処理',   value: 0 },
  { label: '処理中',   value: 1 },
  { label: '完了',     value: 2 },
  { label: 'エラー',   value: 9 },
];
// 表示: 「完了」 → 送信値: 2
```

**バリエーション:**

| パターン | 説明 |
|---|---|
| 基本（label/value マッピング） | 表示は `label`、送信は `value` |
| 単一選択 | 1つだけ選択 |
| 複数選択（チップ表示） | 複数選択+タグ表示 |
| グループ化 | カテゴリ別にグルーピング |
| 検索可能 (showSearch) | テキスト入力で絞り込み |
| disabled状態 | 非活性 |

**試す内容:**
- 選択後の表示値と内部値（送信値）の確認
- 複数選択時の内部値の形式（配列）
- 検索時のフィルタリング挙動

## 比較観点

各セクションで以下の3つの観点を確認できるようにする。

### 見た目・UX
- デフォルトのスタイル
- フォーカス時の挙動
- アニメーション
- エラー表示のデザイン

### バリデーション
- 入力値の検証タイミング（onChange / onBlur）
- エラーメッセージの表示方法
- 無効な入力の防止方法

### カスタマイズ性
- フォーマットの変更しやすさ
- スタイルの上書きのしやすさ
- 独自ルールの追加しやすさ

```json
    "time":         { "display": "14:30",            "value": "14:30:00" },
    "number":       { "display": "1,234,567",        "value": 1234567 },
    "text":         { "display": "テスト",            "value": "テスト" },
    "autocomplete": { "display": "はい",              "value": 2 },
    "select":       { "display": "完了",              "value": 2 }
  },
  "antd": {
    "date":         { "display": "2026/03/11",       "value": "2026-03-11" },
    "dateTime":     { "display": "2026/03/11 14:30", "value": "2026-03-11T14:30:00" },
    "time":         { "display": "14:30",            "value": "14:30:00" },
    "number":       { "display": "1,234,567",        "value": 1234567 },
    "text":         { "display": "テスト",            "value": "テスト" },
    "autocomplete": { "display": "はい",              "value": 2 },
    "select":       { "display": "完了",              "value": 2 }
  }

```

これにより「ユーザーに見えている値」と「サーバーに送られる値」の違いを一目で確認できる。

## 備考

- 状態管理は React の `useState` のみで十分（外部状態管理ライブラリ不要）
- MUI と Ant Design のスタイルが干渉しないよう CSS の分離に注意
- 各コンポーネントの **表示値（display）** と **送信値（value）** の違いが明確に分かるようにする
- 各セクション内でバリエーション（パターン）は縦に並べて一覧できるようにする
